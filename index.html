Open Source Software (OSS) is software whose source code is freely available for anyone to view, modify, and distribute. It plays a major role in modern computing, powering operating systems, web servers, mobile devices, and cloud platforms.

Open source encourages collaboration, transparency, and innovation. Many of today’s most important technologies are built using open source software.

What is Open Source?

Open source software allows users to access the source code of a program. This means developers can study how the software works, improve it, and share their changes.

According to the Open Source Initiative (OSI), open source software must:

  • Allow free redistribution
  • Provide access to source code
  • Permit modifications and derived works
  • Not discriminate against users or fields of use

History of Open Source

In the early days of computing (1950s–1970s), software was freely shared among researchers. The concept of open source became formalized in the late 1990s.

In 1983, Richard Stallman launched the Free Software Movement and later created the GNU Project.

In 1998, the term "Open Source" was introduced to make free software more business-friendly and practical.

Open Source Licenses

Open source software is released under specific licenses that define how it can be used.

MIT License
The MIT License is a permissive software license originating at the Massachusetts Institute of Technology (MIT)[6] in the late 1980s.As a permissive license, it puts few restrictions on reuse and has high license compatibility.

Apache License 2.0
The Apache License 2.0 is a widely used, permissive open-source software license that grants broad permissions to use, modify, and distribute software, even in proprietary products, while requiring attribution, inclusion of the license text, and clear notice of changes.

GNU GPL
The GNU General Public Licenses (GNU GPL or simply GPL) are a series of widely used free software licenses, or copyleft licenses, that guarantee end users the freedom to run, study, share, or modify the software.

BSD License
The BSD License is a family of permissive, open-source licenses from the Berkeley Software Distribution, allowing broad use, modification, and distribution with minimal restrictions, even in proprietary software, requiring only attribution and copyright notice retention in source/binary forms.

Open Source vs Closed Source

Open Source:

  • Source code available
  • Free or low cost
  • Community-driven

Closed Source:

  • Source code hidden
  • Paid licenses
  • Controlled by one company

Guide to open source contribution

Why Contribute to Open Source?

  • Improve coding and problem-solving skills
  • Gain real-world project experience
  • Build a strong GitHub portfolio
  • Collaborate with global developers
  • Increase chances of internships and jobs

Basic Requirements

  • Basic programming knowledge (HTML, CSS, JavaScript, Python, Java, etc.)
  • Git and GitHub basics
  • Willingness to learn and read documentation

Step-by-Step Open Source Contribution Process

  1. Choose a Project: Find a beginner-friendly open source project on GitHub
  2. Read Documentation: Understand the README, contribution guidelines, and code structure
  3. Fork the Repository: Create your own copy of the project
  4. Clone the Repository: Download the project to your local system
  5. Create a New Branch: Work on a separate branch for your changes
  6. Make Changes: Fix bugs, add features, or improve documentation
  7. Commit Changes: Write clear and meaningful commit messages
  8. Push to GitHub: Upload your branch to your fork
  9. Create a Pull Request (PR): Request maintainers to review and merge your changes

Beginner-Friendly Contribution Ideas

  • Fix typos or grammar in documentation
  • Improve README files
  • Add comments to code
  • Fix small bugs labeled as "good first issue"
  • Improve UI or accessibility
